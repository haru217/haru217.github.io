If you’re writing Shorts one by one, you’ll always feel behind.
The faster workflow is **batching**: plan → outline → write → publish, in blocks.

This post shows a simple Notion setup you can run in about **60 minutes per week**.
It’s built for solo creators who want consistency without burning out.

## The goal (what “batching” really means)

Batching is not “write 10 scripts in one sitting.”

It’s:
- **Decide topics once** (so you don’t re-think every day)
- **Reuse the same structure** (so scripts don’t start from zero)
- **Ship on a schedule** (so you don’t wait for motivation)

In practice, you want a pipeline where every idea moves through clear stages.

## A minimal Notion database (properties that matter)

Create a database called **Scripts** with these fields:

- **Title** (Title)
- **Status** (Select): Idea / Outline / Draft / Filming / Posted
- **Hook** (Text)
- **3 Beats** (Text) — the 3 main points (or steps)
- **CTA** (Text) — one line, optional
- **Notes** (Text) — sources, examples, data
- **Publish Date** (Date)

That’s enough.
Avoid building a “perfect” dashboard first. The dashboard comes after you ship.

## The 5-step weekly batch session (60 minutes)

### Step 1) Capture 10 raw ideas (10 minutes)

Open a fresh page (or an “Ideas” database) and dump:
- problems your audience complains about
- mistakes beginners repeat
- tools you use every day
- “before → after” transformations

Keep each idea as **one sentence**.
Don’t write scripts yet.

### Step 2) Pick 3 winners (5 minutes)

Choose ideas that match at least **two** of these:
- clear audience (“for {who}”)
- clear outcome (“get {result}”)
- clear timeframe (“in {time}”)
- clear constraint (“without {pain}”)

If it’s fuzzy, it’s not a Short.

### Step 3) Write hooks only (10 minutes)

For each of the 3 winners, write 3 hook options.
Use one of these templates:

- **Result in time:** “Get {result} in {time} (without {pain}).”
- **Stop doing this:** “Stop {mistake} if you want {outcome}.”
- **Do this instead:** “Don’t {bad approach}. Do {better approach}.”

Pick the best hook and paste it into the **Hook** field.

### Step 4) Outline with “3 beats” (15 minutes)

Write the body as three beats:

1. **Context (1 sentence):** what’s happening / what we’re solving
2. **Method (2–3 bullets):** your steps or framework
3. **Payoff (1 sentence):** what changes / what to do next

Paste this into the **3 Beats** field.

### Step 5) Turn outlines into drafts (20 minutes)

Now you’re allowed to draft.
Write in a tight format:

- Hook (1 line)
- Beat 1 (1–2 lines)
- Beat 2 (1–2 lines)
- Beat 3 (1–2 lines)
- Optional CTA (1 line)

The draft doesn’t need to be “good.”
It needs to be **filmable**.

## A Notion view that makes this effortless

Create two views in your Scripts database:

1) **This Week** (filter)
- Status is not Posted
- Publish Date is within the next 7 days

2) **Batch Session** (board)
- Group by Status
- Columns: Idea → Outline → Draft → Filming → Posted

Now you can drag cards forward like a conveyor belt.

## The mistake that kills batching (and how to avoid it)

The most common failure mode is making “Status” mean everything.

If you mix:
- writing
- filming
- editing
- posting

…in one generic “In progress” state, you’ll stall.

Keep your pipeline honest:
- Outline is not Draft
- Draft is not Filming
- Filming is not Posted

When you can see where work is stuck, you can fix it.
