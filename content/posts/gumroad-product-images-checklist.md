If your Gumroad page gets views but not sales, your **product images** are often the bottleneck.
People skim. Images do the heavy lifting.

Here’s a simple, repeatable checklist for digital products like **Notion templates, prompt packs, swipe files, and cheat sheets**.

## The 3-image set that converts

### Image 1) Outcome screenshot ("what I get")
Goal: make the product instantly legible.

Use one clean screenshot that shows the *end state*:
- Notion dashboard / database view
- PDF cover + a zoomed-in snippet
- CSV preview (only if it’s the main deliverable)

**Checklist**
- No personal info in the sidebar
- Large readable text (zoom in)
- No clutter: crop tight
- If it’s Notion: use a public view when possible

### Image 2) What’s included ("exact deliverables")
Goal: remove uncertainty and reduce refunds.

Turn the deliverables into a visual list.
Example:
- 1 Notion template page
- 50 hook lines (CSV + Notion)
- 20 prompts (Markdown)

**Checklist**
- Counts match the actual files/pages
- Use plain words (not internal file paths)
- Avoid vague claims like “tons of resources”

### Image 3) How it works ("3 steps")
Goal: show the setup friction is low.

A simple graphic is enough:
1) Buy
2) Duplicate / download
3) Follow the checklist

**Checklist**
- Mention setup time (e.g., “2 minutes”)
- Mention requirements (e.g., “Free Notion account”)
- If there’s a duplicate link: show it clearly

## Quick formatting rules (that save you)

- Use **3–6 images max**. More = confusion.
- Keep a consistent style: same font, same margins.
- Prioritize clarity over “design.”
- Don’t put your price as a huge badge in the image (prices change).

## A simple order to test

If you don’t know what to choose, try this order:
1) Outcome screenshot
2) What’s included
3) How it works
4) FAQ snippet (optional)
5) Social proof (optional)
