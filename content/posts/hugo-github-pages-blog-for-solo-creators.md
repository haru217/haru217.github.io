If you sell digital products (Notion templates, prompt packs, cheat sheets), your biggest long-term growth lever is **owned traffic**.
A lightweight blog gives you a permanent home to rank on Google—without ads.

This is the simplest stack I recommend:
- **Hugo** (static site generator): fast, simple Markdown writing
- **GitHub Pages**: free hosting
- **A clean theme** (like PaperMod): great readability + performance

## Why Hugo (instead of WordPress)
WordPress works, but you pay the “maintenance tax”:
- updates
- plugins
- security
- slower performance

Hugo is:
- *very* fast
- no database
- easy to version control

If your goal is SEO content volume, Hugo is frictionless.

## The minimal setup (high level)
You only need:
1) a Hugo site repo
2) a theme (as a Git submodule)
3) a GitHub Actions workflow to build + deploy

That’s it.

## SEO basics to set on day 1
Do these early so you don’t rewrite later.

### 1) Human-readable URLs
Keep slugs short and keyword-forward:
- ✅ `/gumroad-product-description-template/`
- ❌ `/post-2026-03-13-1/`

### 2) One topic cluster per product line
Example cluster for a Notion template brand:
- Gumroad SEO
- Notion template onboarding
- YouTube Shorts scripting

Don’t write random topics.
SEO rewards topical focus.

### 3) Internal links like a product funnel
Every article should link to:
- 1 related article (same cluster)
- 1 relevant product page (soft CTA)

You’re not “blogging”. You’re building a path.

## Content workflow that actually ships
Write in this order:
1) **Problem** (what the reader is stuck on)
2) **Quick wins** (3–7 bullets)
3) **Template / checklist** (copy-pastable)
4) **Soft CTA** (link to your product)

If you can’t add a checklist or template, the post is usually too vague.

## What to publish first (3 posts)
If you’re starting from zero, publish these three:
1) a *“best practices”* post (evergreen)
2) a *template* post (high intent)
3) a *mistakes* post (high relevance)

Then expand each into a cluster.
