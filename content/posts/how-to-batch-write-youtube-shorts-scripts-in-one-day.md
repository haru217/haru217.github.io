## TL;DR
- Pick **one micro-topic** and one promise ("get X result in Y time")
- Write **10 hooks first**, then expand into **10 script outlines**, then draft full scripts
- Use a **single repeatable script format** (Hook → 3 bullets → Next step)
- Timebox everything (hooks 20 min, outlines 30 min, drafts 60 min)

## The problem
Writing Shorts scripts one-by-one feels like you’re starting from zero every time.

You open a blank page, try to be original, rewrite the hook five times, and burn 30 minutes… for one script.

Batching fixes that because your brain stays in the same context:
- same audience
- same topic
- same examples
- same tone

## The quick answer
Batch-write Shorts scripts in three passes:

1) **Hooks only** (10–20 hooks)
2) **Outlines only** (turn each hook into 3 bullets)
3) **Drafts only** (fill the bullets into a 25–45 second script)

Don’t mix the passes. Switching modes is what slows you down.

## Step-by-step
### Step 0: Set the target (2 minutes)
Decide:
- **Audience**: who is this for?
- **Promise**: what result do they want?
- **Angle**: one micro-topic you can cover from multiple sides

Example:
- Audience: new freelancers
- Promise: write proposals faster
- Angle: simple proposal structure

### Step 1: Generate 10 hooks (20 minutes)
Use this template:

- **[Result] in [Time] for [Who]**
  - “Write a client proposal in 10 minutes (even if you hate writing)”
- **Stop doing X, do Y instead**
  - “Stop writing long proposals. Use this 3-line format.”
- **If you’re doing X, you’re losing Y**
  - “If your proposal starts with your bio, you’re losing clients.”

Rule: hooks must be **one sentence**.

### Step 2: Convert hooks into outlines (30 minutes)
For each hook, write:
- Bullet 1: the mistake / problem
- Bullet 2: the fix (principle)
- Bullet 3: a concrete example

Keep bullets short. You’re building a production line.

### Step 3: Draft the scripts (60 minutes)
Use this 30–45 second format:

1) Hook (1 line)
2) 3 bullets (1 sentence each)
3) Next step (1 line)

**Next step ideas** (low-friction):
- “Save this and try it on your next video.”
- “Comment ‘template’ and I’ll reply with the format.”
- “If you want the full workflow, link is in the description.”

### Step 4: Quality pass (10 minutes)
Skim every script and check:
- first line has a clear promise
- no long setup
- each bullet is concrete
- total read time fits your target length

## Common mistakes
- **Trying to perfect the first script** → batch first, polish later.
- **Switching between hook-writing and drafting** → you’ll slow down.
- **Changing the format every time** → consistency is the point.
- **Writing about multiple topics in one batch** → keep one micro-topic.

## Template / checklist
Copy/paste this checklist into your notes:

- [ ] Choose audience + promise + micro-topic
- [ ] Write 10–20 hooks (one sentence each)
- [ ] Turn each hook into 3 bullets (mistake → fix → example)
- [ ] Draft scripts using Hook → 3 bullets → Next step
- [ ] Quick skim (promise, concrete bullets, length)
